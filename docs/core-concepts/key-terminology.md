# Key Terminology

A glossary of terms used throughout BlockAI documentation.

## Platform Terms

| Term                     | Definition                                                                                     |
| ------------------------ | ---------------------------------------------------------------------------------------------- |
| **Command Center**       | BlockAI's unified dashboard — the central hub for all tools and data                           |
| **AI Credits**           | The currency used to access AI-powered features. Each action costs 1–3 credits                 |
| **Chain Scanner**        | BlockAI's wallet analysis module — scans addresses and decodes transactions                    |
| **Smart Alerts**         | Configurable notifications triggered by on-chain events, price changes, or gas conditions      |
| **Controlled Execution** | The flow of generating → auditing → deploying a smart contract with user approval at each step |
| **Daily Rewards**        | Streak-based point system rewarding daily dashboard visits                                     |
| **Access Gateway**       | The feature-gating system requiring an access code for beta features                           |

## Blockchain Terms

| Term                 | Definition                                                                                               |
| -------------------- | -------------------------------------------------------------------------------------------------------- |
| **EVM**              | Ethereum Virtual Machine — the runtime environment for smart contracts on Ethereum and compatible chains |
| **ERC-20**           | Token standard for fungible tokens on EVM chains                                                         |
| **ERC-721**          | Token standard for non-fungible tokens (NFTs)                                                            |
| **Gas**              | The fee required to execute transactions on blockchain                                                   |
| **Wallet Address**   | A unique identifier (0x...) representing an account on an EVM chain                                      |
| **Transaction Hash** | A unique identifier for a specific blockchain transaction                                                |
| **TVL**              | Total Value Locked — the total capital deposited in a DeFi protocol                                      |
| **DeFi**             | Decentralized Finance — financial applications built on blockchain                                       |
| **Layer 2 (L2)**     | Scaling solutions built on top of Ethereum (e.g., Base, Arbitrum, Optimism)                              |

## Technical Terms

| Term         | Definition                                                        |
| ------------ | ----------------------------------------------------------------- |
| **JWT**      | JSON Web Token — the authentication token format used by BlockAI  |
| **Prisma**   | The ORM (Object-Relational Mapping) used in BlockAI's backend     |
| **ThirdWeb** | Wallet infrastructure SDK used for multi-wallet connection        |
| **ChainGPT** | AI model provider specializing in blockchain-specific AI services |
| **Solidity** | The programming language for Ethereum smart contracts             |
| **REST API** | The interface pattern used for all BlockAI backend endpoints      |
